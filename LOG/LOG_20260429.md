## タスク: pyttsx3 TTS サンプルコード作成

### 目的
PLAN.md の音声対話アプリ「talk_with_me」において、pyttsx3 による読み上げがどんな感じか確認するためのサンプルコードを作成。

### 実施内容
1. **venv 環境構築**: `python -m venv venv` で仮想環境を作成
2. **pyttsx3 インストール**: `pip install pyttsx3` (v2.99) — 依存パッケージ: comtypes, pypiwin32, pywin32
3. **音声一覧確認** (`check_voices.py`):
   - Microsoft Haruka Desktop - Japanese (`ja-JP`) ← 日本語音声あり ✅
   - Microsoft Zira Desktop - English (United States) (`en-US`)
4. **サンプルコード作成** (`sample_tts.py`):
   - デモ1: シンプル読み上げ (engine.say + runAndWait)
   - デモ2: 別スレッド版 (PLAN.md の Speaker 設計を再現: queue.Queue + is_speaking フラグ)
   - デモ3: 読み上げ速度比較 (rate 120/150/180/220)
5. **requirements.txt 生成**

### 作成ファイル
| ファイル | 内容 |
|---|---|
| `sample_tts.py` | pyttsx3 TTS サンプル (3種のデモ + 対話メニュー) |
| `check_voices.py` | 利用可能な音声一覧を表示する簡易スクリプト |
| `requirements.txt` | pip freeze で生成 |
| `venv/` | Python 仮想環境 |

### 確認結果
- pyttsx3 は Windows SAPI5 経由で動作し、Haruka (日本語) 音声が利用可能
- サンプルコード `sample_tts.py` を `python sample_tts.py` で実行し、メニューから各デモを試せる状態
- 速度比較テストで rate 120/150/180/220 を試聴し、**rate=150 に決定** (ユーザ承認済み)
- pyttsx3 の癖: `getProperty("rate")` は常にデフォルト値 200 を返す（`setProperty` は正しく機能する）
- 同一エンジンインスタンスで rate を変更しても反映されない場合がある → エンジン再作成で対応

### PLAN.md 更新
- TTS セクションに「✅ ユーザ承認済み」を追記

## タスク: 長文発話のユーザ承認機能

### 実施内容
1. `CONFIRMATION_THRESHOLD = 10.0` 秒 — 10秒超の発話でトリガー
2. `record_speech()` が `(bytes, float)` タプルを返すように変更（発話秒数を取得）
3. 承認フローの変遷:
   - v1: ターミナルで `input()` → Enter/n で承認/破棄 → **EOFError で無承認送信のバグ**
   - v2: EOFError 時はデフォルト破棄に修正 → **ターミナル操作が必要な問題**
   - v3: `voice_draft.md` に書き出し → ユーザが VS Code で編集 → ターミナルで Enter → **まだ input() 依存**
   - v4 (最終): `voice_draft.md` に書き出して停止。`input()` 完全排除。ユーザが手動で `voice_input.md` に保存すれば送信
