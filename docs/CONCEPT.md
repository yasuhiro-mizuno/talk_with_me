# talk_with_me — コンセプト

## 何をするアプリか

**Cline（VS Code のコーディングエージェント）を音声で操作する**ための音声入出力ブリッジ。

ユーザが声で指示を出すと、その内容が Cline のチャットに入力される。Cline がファイル編集やコマンド実行をした後の応答テキストは、音声で読み上げられる。キーボードを使わずに Cline と対話できるようにするのがゴール。

## ユースケース

- VS Code を開きながら、声で「このファイルのここを直して」と指示 → Cline がコードを修正
- 「テスト実行して」と声で指示 → Cline がターミナルでコマンドを実行
- Cline の応答（「修正しました」「エラーが出ました」等）が音声で読み上げられる
- チャット入力の代わりに声を使うだけ。Cline の機能（LLM 呼び出し、ファイル操作、コマンド実行）はそのまま活用

## このアプリの役割

```
┌─────────┐        ┌──────────────┐        ┌─────────┐
│  ユーザ  │──声──▶│ talk_with_me │──文字──▶│  Cline  │
│  (マイク) │       │ (音声ブリッジ) │        │ (VS Code) │
│  (スピーカー)│◀──音声──│              │◀──文字──│         │
└─────────┘        └──────────────┘        └─────────┘
```

- **入力**: マイクで音声を拾い → 文字起こし → Cline のチャットに送信
- **出力**: Cline の応答テキストを取得 → 音声で読み上げ
- **やらないこと**: LLM 呼び出し、ファイル編集、コマンド実行（すべて Cline の仕事）

## ペルソナ

ペルソナは Cline 側（Claude）の設定に依存する。このアプリは音声の入出力変換のみを行い、AI の振る舞いには関与しない。

## 技術スタック概要

| 機能 | 採用技術 | 備考 |
|------|---------|------|
| **音声認識 (STT)** | faster-whisper (`small`) + PyAudio + Silero VAD | マイク録音→音声区間検出→文字起こし。ローカル・オフライン ✅承認済み |
| **音声読み上げ (TTS)** | pyttsx3 (Windows SAPI5, Haruka) | オフライン無料、rate=150 ✅承認済み |
| **Cline 入力連携** | 検討中（クリップボード + キーシミュレーション / VS Code API） | |
| **Cline 出力取得** | Cline rule + ファイル監視 ✅実装済み | Cline が `voice_summary.md` に1行サマリを書く → アプリが監視・読み上げ |
| **画面 (UI)** | ターミナル表示 (rich) | ステータス表示 |

## 動作イメージ

1. VS Code で Cline を開いた状態でアプリ起動
2. 🎤 常時リスニング開始
3. ユーザが「この関数のエラーハンドリングを追加して」と話す
4. faster-whisper で文字起こし → Cline のチャット入力欄にテキスト送信
5. Cline が Claude に問い合わせ → コード修正を実行 → 応答テキストを返す
6. 応答テキストを pyttsx3 で読み上げ (🗣)
7. 読み上げ完了 → 🎤 に戻る

## 自己音声ループ防止

TTS で読み上げ中にマイクが自分の声を拾って暴走するのを防ぐため、読み上げ中は `is_speaking` フラグを立て、STT の認識結果を破棄する。

## 対象環境

- **OS**: Windows 10/11
- **VS Code**: Cline 拡張がインストール済み
- **GPU**: 不要 (CPU のみで動作)
- **マイク**: 必須
- **インターネット**: Cline の Claude API 通信に必要。STT モデルは初回のみダウンロード

## 主要な技術課題

1. **Cline への入力方法** ⬜ 未解決: チャット入力欄にどうやってテキストを送るか
   - クリップボード + キーシミュレーション（pyautogui 等）
   - VS Code Extension API
2. **Cline の応答取得** ✅ 解決済み: Cline rule で `voice_summary.md` に1行サマリを書かせ、アプリがファイル監視して読み上げ
   - Cline のチャット応答を直接解析する必要がなくなった
   - 長文・コード混在の問題も解決（Cline 自身が話し言葉で1文に要約する）

## スコープ外

- 独自の LLM 呼び出し（Cline に任せる）
- ファイル編集・コマンド実行の直接実装（Cline に任せる）
- GUI
- モバイル対応
