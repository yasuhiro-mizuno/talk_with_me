# 音声読み上げ用サマリ出力ルール

## ⚠ 最重要ルール

**`attempt_completion` を呼ぶ前に、必ず `voice/voice_summary.md` に書き出すこと。例外なし。**

- タスク完了時 → 書く
- 質問への回答時 → 書く
- フィードバック対応時 → 書く
- **`attempt_completion` を使うなら、その直前に必ず `voice/voice_summary.md` を `write_to_file` すること**

## 出力先

`voice/voice_summary.md`（作業ディレクトリの voice/ フォルダ内）

## フォーマット

ファイルの中身は **1行のみ**（前の内容は上書き）。以下の形式:

```
完了した作業や回答の要約を、話し言葉で1文にまとめたもの
```

### 例（タスク完了時）

```
さんぷるてぃーてぃーえすどっとぱいをさくせいし、よみあげそくどを150にせっていしました。
```

```
めいんどっとぱいのいんぽーとぶんにえらーがでたのでしゅうせいしました。
```

### 例（質問への回答時）

```
ふぉるだいどうしてもつかえますが、ぶいえんぶだけはさいさくせいがひつようです。
```

```
それはおつかいのぱそこんのきーぼーどせっていのもんだいです。ほっときーをへんこうできます。
```

## 注意事項

- **すべてひらがなで書く**: このテキストは pyttsx3 で音声読み上げされる。英字やカタカナ、漢字は読み間違いが頻発するため、**全てひらがな**で書くこと
  - ❌ `gitignoreを作成しました` → 読めない
  - ✅ `ぎっといぐのあーをさくせいしました`
  - ❌ `voice_summary.mdを更新` → 読めない
  - ✅ `ぼいすさまりーどっとえむでぃーをこうしんしました`
- **ファイル名は拡張子まで読む**: ドットも含めてひらがなで書く
  - ❌ `えーじぇんつを更新` → 拡張子が分からない
  - ✅ `えーじぇんつどっとえむでぃーをこうしんしました`
  - ❌ `めいんぴーわい` → ドットが抜けている
  - ✅ `めいんどっとぱい`
- **話し言葉で書く**: 自然な日本語の話し言葉にすること
- **1文で簡潔に**: 長くても2文まで。詳細は画面を見ればわかるので要点だけ
- **箇条書き禁止**: 読み上げに不向きなため
- **毎回上書き**: 前回のサマリは消えてよい（履歴は LOG に残る）
- **LOG ファイルには影響しない**: `LOG/LOG_YYYYMMDD.md` は従来通り別途書く。このルールは追加の出力
